Mengenal 413 Request Entity Too Large

  • Error 413 Request Entity Too Large * adalah kesalahan sistem yang terjadi ketika kita mengupload file berukuran besar ke server melebihi standar dari yang ditetapkan dalam pengaturan webserver atau penyedia layanan webhosting.

Suatu webserver atau penyedia hosting biasanya telah mempunyai pengaturan terkait besar file yang di izinkan untuk diupload ke server. Pesan ini muncul saat kita mengupload file dengan ukuran yang melebihi ketentuan yang telah ditetapkan oleh webserver atau pengelola layanan hosting.

Mengubah Pengaturan Webserver

Untuk menangani masalah tersebut, kita bisa mengubah pengaturan Nginx terkait besar file yang diperbolehkan untuk di upload.

Buka file konfigurasi nginx dengan perintah berikut :

vi /etc/nginx/nginx.conf

atau

nano /etc/nginx/nginx.conf

kemudian tambahkan configurasi berikut kedalam block http,

        # set client body size to 16M #
        client_max_body_size 16M;
}

client_max_body_size berfungsi untuk menetapkan besaran file yang diperbolehkan untuk diupload. Dari contoh diatas maka block http pada file nginx.conf telah diubah menjadi seperti berikut ini

http {

        ##
        # Basic Settings
        ##

        # set client body size to 16M #
        client_max_body_size 16M;

        ...
}

Dalam konfigurasi diatas, kita menetapkan besar file maksimal yang diperbolehkan untuk diupload ke server sebesar 16 MB.

Bagaimana jika file yang ingin anda upload besar nya 32MB ? anda tinggal mengganti baris client_max_body_size 16M menjadi client_max_body_size 32M

Setelah itu kita akan menerapakan konfigurasi diatas dengan mereload nginx menggunakan perintah,

sudo nginx -s reload

atau

sudo systemctl restart nginx

Sampai disini, silahkan coba upload ulang file anda.
